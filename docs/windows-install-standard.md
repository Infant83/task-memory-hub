# Windows Install Standard

Updated: 2026-05-03

## Decision

TMH currently supports a Python package install with Windows command shims and startup registration scripts. It does not yet ship a standalone MSI/MSIX/NSIS installer or a self-contained single-file desktop app.

## What Exists Today

After:

```powershell
python -m pip install -e ".[tray]"
```

Python creates Windows executable shims such as:

- `tmh.exe`
- `tmh-web.exe`
- `tmh-worker.exe`
- `tmh-mcp.exe`
- `tmh-tray.exe`

These are real `.exe` launcher files in the active Python environment's `Scripts` directory, but they are not standalone application binaries. They still require the Python environment and installed package.

## Standard Install Path

Use this for local development and current on-prem pilots:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\install-tmh.ps1 -RegisterHubStation -StartNow -DesktopShortcuts
```

This script:

1. checks Python >= 3.12,
2. installs TMH with tray extras,
3. checks `tmh`, `tmh-web`, `tmh-worker`, `tmh-mcp`, and `tmh-tray`,
4. attempts to install the BurntToast PowerShell module for current-user toast notifications,
5. creates clickable Start Menu shortcuts,
6. optionally creates Desktop shortcuts,
7. registers Hub Station startup through Task Scheduler,
8. falls back through the existing Hub Station installer if needed,
9. optionally starts the tray/API/worker station immediately.

Double-click wrapper:

```text
installers\TaskMemoryHub-Install.cmd
```

## Shortcut Standard

The install script creates this Start Menu folder:

```text
Start Menu\Programs\Task Memory Hub
```

It contains:

| Shortcut | Purpose |
|---|---|
| `Start TMH Hub Station` | Starts the tray station, loopback Web UI, and worker dispatcher. |
| `Open TMH Web UI` | Opens `http://127.0.0.1:8787/`. |
| `Quick Add Task` | Opens the browser quick-add page. |
| `API Docs` | Opens Swagger UI at `/docs`. |
| `Hub Station Status` | Opens a PowerShell status window for scheduled task, process, port, shortcuts, and API health. |
| `Stop TMH Hub Station` | Stops the tray station process. |

When `-DesktopShortcuts` is passed, the install script also creates:

- `Task Memory Hub Station`
- `Task Memory Hub Web UI`

The standard tray execution path is:

1. click `Start TMH Hub Station`,
2. confirm the tray icon appears,
3. click `Open TMH Web UI` or browse to `http://127.0.0.1:8787/`,
4. use `Hub Station Status` if the tray icon or Web UI is not visible.

## Toast Standard

TMH notification dispatch uses `toast-fallback` by default.

- If BurntToast is available, Windows toast is attempted.
- If BurntToast is missing or fails, TMH writes the notification to the local notification log.
- Install should not fail only because toast is unavailable unless `-RequireToast` is explicitly passed.

Manual BurntToast install:

```powershell
Install-Module BurntToast -Scope CurrentUser -Force -AllowClobber
```

## Startup Standard

Preferred startup path:

1. current-user Scheduled Task at logon,
2. Startup folder VBS fallback when Scheduled Task registration fails.

Status check:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\status-tmh-hub-station.ps1
```

Uninstall:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\uninstall-tmh-hub-station.ps1 -StopNow
```

By default, uninstall removes the scheduled task, startup-folder launcher, Start Menu shortcuts, and known Desktop shortcuts. Use `-KeepShortcuts` only for debugging.

Double-click wrapper:

```text
installers\TaskMemoryHubStation-Uninstall.cmd
```

## Future Installer Track

The stable release path should be staged:

| Stage | Output | When to use |
|---|---|---|
| P0 | PowerShell bootstrap + Python shims | Current local-first development |
| P1 | Signed zip release with bootstrap script | Public MVP download |
| P2 | PyInstaller-built `tmh-tray.exe` and CLI executables | Python-free operator machine |
| P3 | MSI/MSIX/NSIS installer | Non-developer install and enterprise deployment |

Do not jump to P3 before tray lifecycle, logs, upgrades, uninstall, and data migration are stable.

## Common Misunderstandings

| Question | Standard answer |
|---|---|
| Is there a TMH installer `.exe` today? | No standalone installer exists yet. |
| Why do I see `tmh.exe`? | It is a Python entrypoint launcher generated by package install. |
| Does toast require admin? | Normally no. BurntToast can install under `CurrentUser`, but PowerShell repository/policy/network restrictions can block it. |
| Does startup require admin? | Current-user Scheduled Task usually does not. If it fails, TMH uses Startup folder fallback. |
| Is SQLite installed separately? | No. SQLite is built into Python's standard library for this app stage. |
| Is PostgreSQL required? | No. PostgreSQL remains a later migration target. |
